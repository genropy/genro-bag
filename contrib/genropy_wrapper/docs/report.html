Ottimizzazioni Applicate

genro-bag: _htraverse fast path

Principio: in _htraverse, se il path non contiene . (single segment), restituisce (self, path) immediatamente, bypassando la pipeline completa (_htraverse_before → smartsplit → _traverse_inner → smartcontinuation). Questo copre chiavi semplici, #N, #attr=val e key?attr. Tutti i chiamanti (get_item, set_item, pop, get_node, set_attr, __contains__) ne beneficiano senza duplicazione di logica.

genro-bag: altre ottimizzazioni

InterventoDettaglio
Rimozione cast()Rimossi 20 cast() da 6 file. Ogni cast() costava ~44 ns a runtime. Risolto anche un NameError in _query.py (BagNode non definito a runtime con TYPE_CHECKING).

Conclusione

Il wrapper replacement.gnrbag e un drop-in replacement completo di gnr.core.gnrbag. 687 test analysis + 1588 test core, tutti verdi.

Profilo prestazionale favorevole nei casi d'uso tipici Genropy: accesso scalare 1.5x piu veloce, set existing 2.4x piu veloce, creazione bag 2–8x piu veloce, lookup/contains su bag medie-grandi fino a 18x piu veloce, serializzazione XML/JSON 1.2–4.5x piu veloce.

Le operazioni piu lente (nested path 2.2x, positional 2.0x, set+pop 1.6x) hanno overhead assoluto < 3 µs/op, non percepibile in uso reale. Il pickle (1.8x) e a basso volume. La migrazione puo procedere in modo sicuro e incrementale.
